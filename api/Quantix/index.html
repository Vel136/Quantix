Quantix

Deterministic, phase-ordered stat modifier system for Roblox Luau.

Quantix manages numeric and table-based stats through a typed modifier pipeline. Modifiers are sorted into phases (SetBase → FlatAdd → AddPercent → Multiply → Override → Clamp → Lock), evaluated in a fixed order, and cached until invalidated.

Behavior modifier types (BehaviorOverride, BehaviorDeepMerge, BehaviorHook, BehaviorExclusive, BehaviorReplace) run through the same pipeline, replacing ad-hoc merge logic for table-based stats. 

local Quantix = require(ReplicatedStorage.Quantix)

local stats = Quantix.new({
    Damage  = 25,
    Range   = 50,
    Spread  = { Base = 1.5, ADS = 0.8 },
})

local id = stats:AddModifier({
    Stat   = "Damage",
    Type   = Quantix.Types.FlatAdd,
    Value  = 10,
    Source = Quantix.Sources.Attachment,
})

print(stats:Get("Damage"))   --> 35
stats:RemoveModifier(id)
print(stats:Get("Damage"))   --> 25

Signals​

</>
Quantix.Signals: StatControllerSignals

Lifecycle signals for this controller.

Signal Parameters Fires when
OnStatChanged statName: string, newValue: any, oldValue: any A stat's final value changed
OnModifierAdded modifier: Modifier A modifier was added
OnModifierRemoved modifierId: string A modifier was removed
OnBatchEnd (none) A Batch completed 
stats.Signals.OnStatChanged:Connect(function(statName, newValue, oldValue)
    print(statName, oldValue, "→", newValue)
end)

Functions​

new​

</>
Quantix.new(
data: table--

Weapon/entity data table. Pass {} to register stats manually.

) → StatController

Creates a new StatController. Numeric leaf values in data are flattened into dot-separated stat keys. 

-- Nested table → flat keys
local stats = Quantix.new({
    Damage = 25,
    Spread = { Base = 1.5, ADS = 0.8 },
})
-- stats:Get("Spread.Base") → 1.5

-- Manual registration
local stats = Quantix.new({})
stats:SetBase("Damage", 25)

Get​

</>
Quantix:Get(
statName: string--

The stat key to evaluate.

) → any--

The final value, or nil if the stat has no base.

Returns the final evaluated value of a stat. Result is cached until the stat is invalidated. Conditional stats are always re-evaluated. 

print(stats:Get("Damage"))        --> 35
print(stats:Get("Spread.Base"))   --> 1.5

Quantix:GetBase(statName: string) → any

Returns the raw base value of a stat before any modifiers are applied.

print(stats:GetBase("Damage"))   --> 25

</>
Quantix:GetAll() → {[string]: any}

Evaluates all registered stats and returns them as a { [statName]: value } table.

local all = stats:GetAll()

GetGroup​

</>
Quantix:GetGroup(
prefix: string--

The dot-path prefix to filter by.

) → {[string]: any}

Returns all stats whose keys start with prefix., with the prefix stripped.

-- Given stats: Spread.Base = 1.5, Spread.ADS = 0.8
local spread = stats:GetGroup("Spread")
print(spread.Base, spread.ADS)   --> 1.5   0.8

AddModifier​

</>
Quantix:AddModifier(
modifier: Modifier--

The modifier to add. Id is assigned automatically.

) → string--

The auto-generated modifier ID.

Adds a modifier to the controller. Returns the modifier's auto-generated ID, which can be passed to RemoveModifier later.

Required fields: Stat, Type, Value.

local id = stats:AddModifier({
    Stat      = "Damage",
    Type      = Quantix.Types.FlatAdd,
    Value     = 10,
    Source    = Quantix.Sources.Attachment,
    SourceId  = attachment.Id,
    Priority  = 0,
})

RemoveModifier​

</>
Quantix:RemoveModifier(modifierId: string) → boolean

Removes a modifier by its ID. Returns true if the modifier was found and removed, false if it did not exist. 

local removed = stats:RemoveModifier(id)

RemoveBySource​

</>
Quantix:RemoveBySource(
source: string,--

The source category to match.

sourceId: (string | number)?--

Optional specific source instance to match.

) → number--

The number of modifiers removed.

Removes all modifiers with the given Source (and optionally SourceId). Runs inside an implicit Batch. Returns the number of modifiers removed. 

-- Remove all modifiers from any attachment with this ID
stats:RemoveBySource(Quantix.Sources.Attachment, attachment.Id)

-- Remove all Perk modifiers regardless of SourceId
stats:RemoveBySource(Quantix.Sources.Perk)

ReplaceBySource​

</>
Quantix:ReplaceBySource(
source: string,
sourceId: (string | number)?,
newModifiers: {Modifier}
) → {string}--

IDs of the newly added modifiers.

Removes all modifiers matching source/sourceId, then adds newModifiers atomically inside a single Batch. Returns a list of the new modifier IDs. 

local ids = stats:ReplaceBySource(
    Quantix.Sources.Attachment,
    old.Id,
    newAttachment.Modifiers
)

Unique key for this behavior, used as the Stat field in modifiers.

Evaluate​

</>
Quantix:Evaluate(
behaviorKey: string,--

The behavior key (must be registered via RegisterBehavior).

baseBehavior: table--

Fallback if no modifiers have been applied yet.

) → (
table,--

The final merged behavior table.

table--

Hooks table: { [eventName] = { fn, ... } }.

)

Evaluates behavior modifiers for behaviorKey and returns the merged behavior table and collected hooks. 

local behavior, hooks = stats:Evaluate("Bullet", baseBehavior)
-- behavior: merged table (no _Hooks key)
-- hooks: { [eventName] = { fn, fn, ... } }

for _, fn in hooks.OnPierce or {} do
    fn(ctx, result, velocity)
end

Batch​

</>
Quantix:Batch() → ()

Runs fn inside a batch. All OnStatChanged, OnModifierAdded, and OnModifierRemoved signals are deferred until fn returns, then fired once per changed stat. OnBatchEnd fires last. 

stats:Batch(function()
    stats:RemoveBySource(Quantix.Sources.Attachment, id)
    stats:AddModifier({ Stat = "Damage", Type = "FlatAdd",  Value = 8,   Source = "Attachment", SourceId = id })
    stats:AddModifier({ Stat = "Range",  Type = "Multiply", Value = 0.1, Source = "Attachment", SourceId = id })
end)

BeginBatch​

</>
Quantix:BeginBatch() → ()

Manually begins a batch. Prefer Batch for automatic error handling.

stats:BeginBatch()
-- ... modifier changes ...
stats:EndBatch()

EndBatch​

</>
Quantix:EndBatch() → ()

Ends the active batch, flushing deferred signals.

for _, mod in stats:GetModifiersForStat("Damage") do

end

Trace​

</>
Quantix:Trace(statName: string) → string

Returns a formatted multi-line string showing the full evaluation pipeline for statName: base value, all active modifiers, per-phase accumulators, and final result. Skipped (condition-failed) modifiers are listed separately. 

print(stats:Trace("Damage"))
-- Stat:  Damage
-- Base:  25
-- Active modifiers: 2
--   [FlatAdd] Attachment/  value=10  priority=0
-- Phase [FlatAdd]:
--   FlatAdd: { sum=10 }
-- = Final: 35

DebugDump​

</>
Quantix:DebugDump() → ()

Prints a full dump of all registered stats and active modifiers to the Roblox console. Useful for in-editor inspection. 

stats:DebugDump()

Quantix:Destroy() → ()

Destroys the controller. Disconnects all signals and clears all internal state. Always call this when the owning entity (weapon, character) is removed. 

stats:Destroy()
